# `/plan`

Use this command only after an active numbered spec exists.

## Required workflow

1. Confirm `projects/<app>/AGENTS.md` points to the active spec.
2. Create or update `PLAN.md` for architecture-sensitive or multi-module work.
3. Confirm `tasks.md` is aligned to the intended delivery sequence.
4. Confirm `workflow-receipts.md` exists and the relevant workflow sections are initialized.

Use `standards/command-workflow-contract.md` and `standards/spec-driven-workflow.md` as the local contract.
