# `/tasks`

Use this command to refine the ordered implementation checklist for the active numbered spec.

Working directory: `projects/<app>/`.

## Required workflow

1. Update the active `tasks.md`.
2. Mark whether UI, data, mobile, or release-readiness workflows are expected.
3. If sensitive work is in scope, update `checklist.md`.
4. Keep `workflow-receipts.md` aligned with the workflow classification.

Do not treat this command as a replacement for the local workflow skills.
